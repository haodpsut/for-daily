## PHẦN V — DÀNH CHO BAN CHỦ NHIỆM KHOA (Không công khai cho GV)

### 5.1 Bảng tổng hợp năng lực GV → Track

> Sau khi thu phiếu, BCN Khoa điền bảng tổng hợp:

| STT | Họ tên GV | Học vị | Môn đang dạy | Điểm TB Track A | Điểm TB Track B | Điểm TB Track C | Nền tảng chung | Track phù hợp nhất | Cần bồi dưỡng |
|-----|-----------|--------|-------------|-----------------|-----------------|-----------------|----------------|-------------------|---------------|
| 1 | | | | | | | | | |
| 2 | | | | | | | | | |
| 3 | | | | | | | | | |
| ... | | | | | | | | | |

**Cách tính điểm TB Track:**
- Track A: Trung bình mục 2.2 (10 câu)
- Track B: Trung bình mục 2.3 (10 câu)
- Track C: Trung bình mục 2.4 (10 câu)
- Nền tảng: Trung bình mục 2.1 (8 câu)

### 5.2 Phân tích Gap (Thiếu hụt)

> Dựa trên kết quả khảo sát, xác định:

| Track | Số GV đủ NL dạy (điểm TB ≥ 3.5) | Số GV cần bồi dưỡng (2.5–3.4) | Thiếu hoàn toàn (< 2.5) | Số GV cần cho Track | Kết luận |
|-------|----------------------------------|-------------------------------|-------------------------|---------------------|---------|
| A — AI & Data | /__ GV | /__ GV | /__ GV | 3–4 GV | ☐ Đủ ☐ Cần bồi dưỡng ☐ Cần tuyển mới |
| B — Web & Mobile | /__ GV | /__ GV | /__ GV | 3–4 GV | ☐ Đủ ☐ Cần bồi dưỡng ☐ Cần tuyển mới |
| C — Network & IoT | /__ GV | /__ GV | /__ GV | 3–4 GV | ☐ Đủ ☐ Cần bồi dưỡng ☐ Cần tuyển mới |
| Nền tảng chung | /__ GV | /__ GV | /__ GV | 5–6 GV | ☐ Đủ ☐ Cần bồi dưỡng ☐ Cần tuyển mới |
